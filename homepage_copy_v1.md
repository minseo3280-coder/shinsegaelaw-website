## 0. 전체 디자인 원칙

- [x] **영문소제목 + 골드라인 + 한글대제목** 반복 패턴 → **섹션마다 완전히 다른 레이아웃**

- [x] 모든 섹션 동일한 py-20/py-28 여백 → **섹션별 다른 호흡** (풀스크린, 타이트, 여유)
- [x] 카운터 4개 나란히 → **비대칭 레이아웃, 숫자 하나를 크게**
- [x] 베이지/크림 톤 반복 → **화이트 + 연그레이 교차, 밝은 톤 유지**
- [x] ~~다크 네이비 배경~~ → **전면 제거. 밝은 배경만 사용**

### 버건디 사용 규칙 (촌스럽지 않게)
```
✅ 허용:
- CTA 버튼 배경 (작은 면적)
- 텍스트 강조 (숫자, 라벨, 링크)
- 얇은 라인/보더 (좌측 바, 밑줄 2px)
- 작은 아이콘/뱃지 색상
- 호버 시 텍스트 컬러 전환

❌ 금지:
- 섹션 전체 배경을 버건디로 채우기
- 큰 면적의 버건디 그라디언트
- 버건디 + 골드 동시 과다 사용
- 버건디 배경 위에 화이트 텍스트 대면적
```

### 타이포그래피
- **대제목**: 48~72px, font-weight 700, line-height 1.2, Gowun Batang(명조) — **큼직하게, #1A1A1A**
- **중제목**: 28~36px, font-weight 600, Pretendard
- **본문**: 16~18px, font-weight 400, line-height 1.8, Pretendard, #555
- **캡션/라벨**: 12~14px, letter-spacing 0.05em, 버건디 or #999

### 인터랙션 원칙
- 스크롤 시 **한 섹션씩 드러나는** reveal (translateY 40px → 0, opacity 0 → 1)
- 카드 호버: **미세한 Y축 이동(-4px) + 그림자 확장** (뻔한 scale 금지)
- 이미지 호버: **밝기 미세 변화 + 오버레이 텍스트 등장**
- CTA 버튼 호버: **배경색 채움 애니메이션** (좌→우 wipe)
- 숫자 카운트업: **스크롤 진입 시 0에서 올라가는 애니메이션**
- 전체적으로 **가볍고 부드러운** 트랜지션 (0.3~0.5s ease-out)

## 18. 개발 시 주의사항

1. **이미지 우선**: 업무분야 카드, 변호사 프로필 등 이미지가 핵심. placeholder 이미지로라도 레이아웃 잡을 것
2. **폰트 사이즈 큼직하게**: 대제목 48~64px. readyforsingle 참고 — 시원시원하게
3. **여백 차별화**: Hero는 풀스크린, Trust Band는 타이트, 나머지는 `py-20~py-32`
4. **호버 효과**: 카드마다 다른 호버 (이미지 카드=그림자확장, 텍스트 카드=Y이동, 아코디언=배경색)
5. **Swiper 설정**: Cases/Lawyers는 FreeMode + 드래그, 후기는 Autoplay
6. **버건디 절제**: 포인트에만! CTA 버튼, 숫자, 라벨, 얇은 라인, 작은 아이콘 — 큰 면적 금지
7. **골드 최소화**: 별점, 번호 포인트에만. 골드 배경/그라디언트 금지
8. **그림자 절제**: `shadow-sm` ~ `shadow-md` 범위. `shadow-2xl` 같은 과한 그림자 금지
9. **보더 활용**: `border-[#EBEBEB]` 얇은 보더로 카드/영역 구분 — 깔끔한 느낌
10. **❌ 다크 배경 사용 금지**: `#1A1A2E`, `#0F0F1A`, `bg-black` 등 어두운 배경 절대 사용 안 함
